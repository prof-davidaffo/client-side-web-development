Drag and Drop API

Impara a rendere un elemento trascinabile, autorizzare una zona di rilascio e spostare nodi nel DOM usando gli eventi nativi del browser.

Obiettivo: costruire una piccola interfaccia in cui l'utente trascina elementi tra contenitori.

Rendere un elemento trascinabile

Usare l'attributo draggable="true" sugli elementi che devono partire con un trascinamento.

Trasportare un riferimento

Scrivere e leggere dati con event.dataTransfer, di solito salvando l'id dell'elemento.

Consentire il drop con preventDefault() e poi spostare l'elemento nel contenitore di destinazione.

Il concetto in breve

Il drag and drop è l'interazione con cui afferri un oggetto sullo schermo e lo rilasci altrove. In HTML non basta muovere il mouse: il browser deve sapere quale elemento può essere trascinato, quali zone accettano il rilascio e cosa fare quando il rilascio avviene.

Schema mentale

Partenza: salvo cosa sto trascinando. Passaggio sopra la zona: autorizzo il drop. Rilascio: recupero il dato e aggiorno la pagina.

Cosa osservare

Quando inizi a trascinare, il browser memorizza l'id del riquadro. Quando passi sopra l'area vuota, la pagina autorizza il rilascio. Quando lasci il mouse, il riquadro viene spostato dentro la nuova area.

Eventi e metodi chiave

Un drag and drop funziona solo se prepari sia l'elemento che parte sia il contenitore che riceve. Segui questi passaggi nell'ordine.

Decidi quale elemento si può trascinare

Aggiungi draggable="true" all'elemento che l'utente deve afferrare. Senza questo attributo, un normale div o p non parte come oggetto trascinabile.

<div id="task-1" draggable="true">Studiare dragstart</div>

Quando parte il trascinamento, salva cosa stai spostando

dragstartHandler non è una funzione già esistente del browser: è un nome scelto da noi. Potresti chiamarla anche iniziaTrascinamento. Diventa utile solo quando la colleghi all'evento dragstart.

L'evento dragstart scatta appena inizi a trascinare. Qui salvi un dato dentro dataTransfer. Di solito salvi l'id dell'elemento, perché poi ti servirà per ritrovarlo nel DOM.

Nel nostro esempio non ci sono due tipi di elemento. C'è un solo elemento trascinato e salviamo una sola informazione: il suo id.

"text/plain" non descrive l'elemento. Descrive il valore che stai mettendo in dataTransfer. Siccome l'id è una stringa, cioè testo normale, usiamo "text/plain".

event.dataTransfer.setData("text/plain", "task-1");

Leggila così: "salva il testo task-1 dentro il trasferimento". Quando fai il drop, chiedi indietro quel testo con getData("text/plain").

const elemento = document.getElementById("task-1");
elemento.addEventListener("dragstart", dragstartHandler);

function dragstartHandler(event) {
  event.dataTransfer.setData("text/plain", event.currentTarget.id);
}

Quando passi sopra una zona, autorizza il rilascio

Anche dragoverHandler è solo un nome scelto da noi. La funzione viene eseguita perché la colleghiamo alla zona con addEventListener("dragover", ...).

Per impostazione predefinita il browser non permette di rilasciare elementi ovunque. Nell'evento dragover devi chiamare preventDefault(): è il segnale che quella zona accetta il drop.

const zona = document.getElementById("area-drop");
zona.addEventListener("dragover", dragoverHandler);

function dragoverHandler(event) {
  event.preventDefault();
}

Quando rilasci, recupera l'id e sposta l'elemento

dropHandler è un'altra funzione definita da noi e collegata all'evento drop. Il browser fornisce l'evento; noi decidiamo che codice eseguire quando quell'evento accade.

L'evento drop scatta nel momento del rilascio. Leggi il dato salvato con getData("text/plain"): devi usare lo stesso tipo che avevi usato in setData(). Poi trovi l'elemento con document.getElementById() e lo inserisci nella zona con appendChild().

zona.addEventListener("drop", dropHandler);

function dropHandler(event) {
  event.preventDefault();
  const id = event.dataTransfer.getData("text/plain");
  event.currentTarget.appendChild(document.getElementById(id));
}

Esempio completo minimo

const elemento = document.getElementById("task-1");
const zona = document.getElementById("area-drop");

elemento.addEventListener("dragstart", dragstartHandler);
zona.addEventListener("dragover", dragoverHandler);
zona.addEventListener("drop", dropHandler);

function dragstartHandler(event) {
  event.dataTransfer.setData("text/plain", event.currentTarget.id);
}

function dragoverHandler(event) {
  event.preventDefault();
}

function dropHandler(event) {
  event.preventDefault();
  const id = event.dataTransfer.getData("text/plain");
  event.currentTarget.appendChild(document.getElementById(id));
}

Da ricordare

dragstartHandler, dragoverHandler e dropHandler sono nomi inventati da noi. Non fai override di funzioni native.

"text/plain" non indica un tipo di elemento HTML. Indica il formato del dato salvato. Nel nostro caso il dato salvato è l'id, e l'id è testo.

Regola pratica: scrivi "text/plain" sia quando salvi sia quando leggi. Se salvi con setData("text/plain", id), recuperi con getData("text/plain").

dataTransfer non sposta da solo l'elemento: serve solo a trasportare un'informazione durante il drag.

appendChild() e il comando che sposta davvero il nodo dentro il nuovo contenitore.

currentTarget indica l'elemento su cui hai collegato il listener. In una zona di drop è spesso più sicuro di target, che potrebbe essere un elemento interno.

Mini progetto da 30 minuti

Realizza a parte una piccola bacheca kanban in un file HTML autonomo.

Consegna

Crea un file chiamato mini-kanban.html con tre colonne: Da fare, In corso, Fatto. Inserisci almeno cinque attività trascinabili nella prima colonna e permetti di spostarle tra tutte le colonne.

Requisiti minimi

  • Ogni attività deve avere un id unico e draggable="true".
  • Durante dragstart salva l'id dell'attività con dataTransfer.setData().
  • Ogni colonna deve accettare il rilascio usando dragover e preventDefault().
  • Nel drop recupera l'id con getData() e sposta l'attività nella colonna scelta.

Estensioni facoltative

  • Aggiungi un effetto visivo quando una card passa sopra una colonna.
  • Aggiungi un contatore di attività per ogni colonna.
  • Impedisci il rilascio fuori dalle colonne.

Obiettivo finale

Il progetto è completo quando puoi aprire mini-kanban.html nel browser e spostare le attività avanti e indietro tra le tre colonne senza errori in console.

Checklist

Errori comuni

Dimenticare preventDefault(): il browser non considera valida la zona di rilascio.

Usare id duplicati: getElementById() restituisce un solo elemento, quindi gli id devono essere unici.

Appendere su target senza controllo: se il drop avviene sopra un figlio, potresti spostare l'elemento nel punto sbagliato. Usa currentTarget quando colleghi listener alla zona.
