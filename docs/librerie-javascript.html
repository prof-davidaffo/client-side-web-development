Sviluppo Web lato client

Chapter 13 Librerie JavaScript

Molti web programmer incontrano gli stessi requisiti di programmazione mentre sviluppano web sites interattivi. Per esempio, molti siti devono gestire e manipolare objects e arrays, mostrare componenti standard (come modal windows o contenuti collapsible) o fare forme simili di complessa DOM manipulation. Poiche’ uno dei principi principali dello sviluppo software e’ il reuse, i developer spesso rendono disponibile ad altri il codice che risolve questi problemi sotto forma di libraries e frameworks. Sono file script rilasciati pubblicamente che puoi scaricare e includere nel tuo project, permettendoti di sviluppare applicazioni complesse piu’ rapidamente e facilmente. Questa e’ la cosa fantastica della comunita’ open-source: le persone risolvono problemi e poi rendono quelle soluzioni disponibili agli altri.

Le web applications moderne fanno un uso esteso di librerie esterne, sia integrando molte librerie diverse in una sola app, sia facendo affidamento su un framework specifico (che puo’ a sua volta essere composto da molte librerie diverse)! Questo capitolo descrive come includere e utilizzare librerie JavaScript esterne nella tua web page, e presenta la popolare libreria jQuery come esempio.

  • Per un’idea di scala, la directory del package manager npm elenca quasi mezzo milione di librerie JavaScript diverse!

Nota che gli script esterni sono in genere chiamati libraries o frameworks. Tuttavia, questi termini non sono completamente intercambiabili. Una library e’ un insieme di behaviors (functions) che puoi usare e chiamare nel tuo codice. Per esempio, Lodash (descritto sotto) e’ una library che fornisce utility functions utilizzabili. Chiami il codice della library quando vuoi. Un framework, invece, fornisce un insieme di codice in cui inserisci i tuoi behaviors, per esempio subclassando componenti forniti o specificando le tue callback functions. Il framework chiama il tuo codice quando vuole lui. Martin Fowler si riferisce a questo come “Inversion of control”. I framework spesso sembrano piu’ facili da usare (fanno di piu’ con meno lavoro da parte tua!), ma possono essere difficili da personalizzare per raggiungere obiettivi precisi. Le libraries possono essere piu’ difficili da usare, ma probabilmente puoi impiegarle esattamente come serve. Questo capitolo si concentra principalmente sulle libraries; React (descritto nei capitoli successivi) e’ piu’ un framework.

13.1 Includere una Library

Proprio come puoi includere piu’ file CSS in una pagina con elementi <link> multipli, puoi includere piu’ script JavaScript con elementi <script> multipli.

<script src="alpha.js"></script> <!-- esegui questo script per primo -->
<script src="beta.js"></script> <!-- esegui questo script per secondo -->

Importante, i file script vengono eseguiti nell’ordine in cui appaiono nel DOM. Questo conta perche’ tutti gli script inclusi tramite l’elemento <script> vengono eseguiti nello stesso namespace. Questo significa che variables e functions dichiarate in uno script sono disponibili negli script successivi—e’ quasi come se tutti i file script fossero combinati in un unico file.

/* alpha.js */
let message = "Ciao Mondo";
/* beta.js */
console.log(message); //=> "Ciao Mondo"
                      //la variable era definita nello script precedente!

L’ordine conta: se beta.js fosse incluso prima di alpha.js, quando viene eseguito la variable message non sarebbe ancora definita, causando un errore.

Altre tecniche per gestire e organizzare grandi numeri di file script saranno discusse nel Chapter 15.

Come per i CSS frameworks, le JavaScript libraries si includono in una pagina fornendo un tag <script> che referenzia il file JavaScript che contiene il codice della library. Per esempio, puoi includere Lodash con utili funzioni di data processing:

<body>
    ...content

    <!-- include JavaScript files -->
    <script src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.4/lodash.min.js"></script>
    <script src="js/my-script.js"></script>
</body>

  • Come per i CSS frameworks, le JavaScript libraries possono essere usate referenziando il file su un CDN, scaricando direttamente il source code (cerca il file .js; potresti dover estrarre un file .zip), oppure installando la library tramite un package manager come npm. Vedi il Chapter 3 per i dettagli—basta sostituire .css con .js!

  • Come per il CSS, i file JavaScript possono essere minimized (di solito chiamati .min.js). Minimizzare i file JavaScript non solo rimuove il whitespace, ma spesso sostituisce anche i nomi delle variables locali con nomi piu’ corti e senza significato come a o x.

Vuoi sempre caricare le libraries esterne prima del tuo script, cosi’ le variables e functions che definiscono saranno disponibili al tuo codice! Se metti prima il tuo script, quelle functions non saranno ancora disponibili!

Dato che tutti i file JavaScript condividono lo stesso namespace, la maggior parte delle libraries popolari rende tutte le sue functions disponibili come properties di una singola global variable. Per esempio, Lodash fornisce una global variable chiamata _ (un singolo underscore—un “low dash”). Usi questa variable per chiamare le functions fornite:

let array = [3,4,3];

let unique = _.uniq(array); //chiama una function dell'object lodash

console.log(unique); //[3,4], gli elementi unici!

  • Una volta caricata la library, i globals che fornisce possono essere accessi come i built-in globals window, document, Math, ecc.

  • Lodash usa un nome di variable apparentemente strano per essere rapido e facile da digitare, e per evitare conflitti con altre variables che potresti creare.

Lodash fornisce molte funzioni utili che possono essere chiamate sull’object globale _. Per esempio:

  • _.random() per generare un numero casuale in un intervallo
  • _.range() per generare un array di numeri in un intervallo
  • _.includes() per determinare se un value e’ in un array (cross-browser!)
  • _.pick() per produrre un nuovo object che ha solo alcune properties di un altro
  • _.merge() per combinare le properties di due objects in un singolo nuovo object

Nota che quale object globale viene fornito (e quali functions ha) e’ diverso per ogni library. Devi guardare la documentazione (spesso il file README.md su Github) per istruzioni ed esempi su come chiamare e utilizzare quella library.

  • “Conoscere” una library o framework significa solo essere familiari con le functions che puoi chiamare!

Pro-tip: Visual Studio Code scarichera’ automaticamente le type definitions per qualsiasi library elencata come dependency nel package.json del progetto. Per esempio, se includi Lodash come dependency (installandola con npm install --save lodash), allora digitando _. VS Code mostrera’ opzioni di autocomplete. Questo rende una buona idea installare le JavaScript libraries via npm, anche se poi pianifichi di caricare la library da un CDN invece che da un file locale.

Se stai usando un linter per controllare il coding style, devi far sapere al linter di qualsiasi global variable caricata in altri script cosi’ il linter non pensi che quelle variables siano undefined. Vedi la documentazione per dettagli su come farlo in ESLint.

13.2 Esempio: jQuery

Come esempio esteso di una library JavaScript, il resto di questo capitolo descrive jQuery – una popolare library JavaScript che aiuta con la DOM manipulation. jQuery e’ stata progettata per fornire metodi piu’ facili, veloci e affidabili per manipolare il DOM nei modi descritti nel capitolo precedente. jQuery e’ una delle librerie piu’ popolari nel web development ed e’ usata da circa il 70% dei siti piu’ popolari.

jQuery e’ stata sviluppata per colmare lacune del linguaggio JavaScript e per standardizzare funzionalita’ tra browser. Ha reso possibile manipolare il DOM scrivendo una frazione del codice, e produrre script che funzionavano sia su Firefox sia su Internet Explorer.

Tuttavia, nei 10+ anni dalla sua uscita, il linguaggio JavaScript e le API DOM sono migliorati (ad es. con querySelector()), cosi’ gran parte di cio’ che rendeva speciale jQuery e’ ora standard e ampiamente supportato. In effetti, l’approccio di jQuery alle pagine interattive e’ diventato pratica comune. Ciononostante, jQuery e’ ancora usata in molti sistemi web (i componenti JavaScript di Bootstrap la usano) ed e’ abbastanza diffusa da rendere utile conoscerla, soprattutto se devi lavorare su un sistema costruito piu’ di 3 anni fa.

Come per altre librerie, includi jQuery referenziandola con un tag <script> nel tuo HTML (prima del tuo script!):

<script src="http://code.jquery.com/jquery-3.2.1.min.js"></script>
  • Nota che la versione jQuery 3.x e’ per i browser attuali. Se devi supportare versioni piu’ vecchie (ad es. IE 6-8), puoi usare la 1.12. Si’, jQuery supporta IE 6!

Caricare la library jQuery crea una variable jQuery nello scope globale, quindi puoi accedervi e usarla nello script.

La variable jQuery e’ in realta’ una function che puoi chiamare, detta Selector function. Questa function seleziona elementi DOM tramite CSS selector, simile a document.querySelector():

//seleziona l'elemento con id="foo" (es. <p id="foo">)
let fooElem = jQuery('#foo');

//seleziona tutti gli elementi <a>
let allLinksArray = jQuery('a');

Tuttavia, e’ molto piu’ comune usare un alias fornito per jQuery(), chiamato $(). Questa scorciatoia permette di selezionare elementi con un solo carattere (invece di 6).

//seleziona l'elemento con id="foo" (es. <p id="foo">)
let fooElem = $('#foo');

//seleziona tutti gli elementi <a>
let allLinksArray = $('a');
  • Come per _ in Lodash, jQuery usa $ perche’ nessuno sano di mente darebbe quel nome a una variable, quindi la probabilita’ di conflitti di namespace e’ minima.

Come document.querySelector(), la function selector di jQuery gestisce la maggior parte dei selector CSS, oltre a qualche pseudo-class utile:

$('#my-div') // per id
$('div') // per tipo
$('.my-class') // per class
$('header, footer') // selector di gruppo
$('nav a') // selector discendente
$('p.red') // selector con scope

$('section:first') // primo elemento <section>
                   // (non e' un css selector!)

Manipolare il DOM

Una volta selezionati gli elementi, jQuery fornisce methods che eseguono la maggior parte delle manipolazioni che faresti con le property del DOM:

var txt = $('#my-div').text();      //ottieni il textContent
$('#my-div').text('new info!');     //cambia il textContent

$('#my-div').html('<em>new html!</em>'); //cambia l'HTML

$('svg rect').attr('height');       //ottieni l'attribute (di tutti i selezionati)
$('svg rect').attr('height',200);   //imposta l'attribute (di tutti i selezionati)
$('svg rect').attr( {x:50, y:60} ); //imposta piu' attributes passando un object

$('section').addClass('container'); //aggiungi class (a tutti i selezionati)
$('section').removeClass('old');    //rimuovi class (da tutti i selezionati)

$('body').css('font-size','24px');  //imposta la property CSS (di tutti i selezionati)
$('body').css( {'font-size':'24px',
                'font-family':'Helvetica'} ); //imposta piu' properties via un object

Se confronti questo con le operazioni equivalenti usando il DOM, noterai che (a) jQuery e’ molto piu’ conciso. jQuery e’ anche piu’ potente: per esempio, puoi impostare piu’ attributes con una singola chiamata a .attr() passando un object che contiene gli attributes da impostare (le keys sono i nomi degli attribute). Questo ti permette di cambiare facilmente molti attributes in una volta!

Importante: nota che tutti questi metodi applicano il cambiamento a tutti gli elementi selezionati dalla funzione selector di jQuery. Non devi usare un loop per applicare cambiamenti a piu’ elementi; puoi semplicemente selezionare gli elementi che vuoi modificare e cambiarli tutti insieme. Questo significa che devi fare attenzione al selector—seleziona solo gli elementi su cui vuoi davvero lavorare!

jQuery fornisce anche metodi che permettono di manipolare il DOM tree (ad es. per aggiungere, spostare o rimuovere elementi):

//crea un elemento (non ancora nel DOM)
let newP = $('<p class="new">Questo e' un nuovo elemento</p>'); //nota il tag!

//aggiungi contenuto al DOM
$('main').append(newP); //aggiungi l'elemento DENTRO <main>, in fondo
$('main').append('<em>nuovo</em>'); //puoi creare un elemento al volo!

$('main').prepend('<em>nuovo</em>'); //aggiungi un nuovo elemento <em> DENTRO <main>, all'inizio

$('main').before('<header>'); //inserisci un nuovo <header> PRIMA di <main> (fratello precedente)
                              //nota che puoi omettere il tag di chiusura se non c'e' contenuto

$('footer').insertAfter('main'); //inserisci il selezionato (<footer>) DOPO il parametro (<main>)
                                 //dato che <footer> era selezionato, si spostera'!

$('main').wrap('<div class="container"></div>'); //circonda <main> con una .container

$('footer').remove(); //rimuove l'elemento <footer> selezionato
$('main').empty(); //rimuove tutti gli elementi child di <main>

La prima cosa importante da notare e’ che crei nuovi elementi passando il contenuto HTML (incluso il tag <>) alla function $(). Questo e’ diverso da document.createElement(), che esplicitamente non include le parentesi angolari <>. Se crei un elemento senza contenuto, puoi anche specificare solo il tag di apertura!

jQuery offre anche metodi di manipolazione piu’ potenti, permettendoti di posizionare elementi facilmente inside, outside, around e in place of altri elementi.

Anche se jQuery ti permette di creare elementi HTML arbitrariamente complessi da aggiungere al DOM, dovresti evitare di scrivere grandi blocchi di HTML dentro il file .js da appendere via jQuery (ad es. non aggiungere un intero DOM tree in una volta!). Questo viola la separation of concerns e rende il codice difficile da leggere, interpretare e modificare (perche’ ora l’HTML e’ in due posti!). Se devi inserire dinamicamente grandi quantita’ di HTML “hard-coded”, scrivi invece quel contenuto nel file .html, rendilo invisibile (ad es. display:none), e poi usa jQuery per spostare e mostrare l’elemento quando serve.

Gestire gli eventi

jQuery fornisce anche convenience methods per registrare event listener:

$('#button').click(function(event) {
   console.log('clic clic!');

   //chi e' stato cliccato
   let element = $(event.target);
});

  • Esistono metodi equivalenti per altri eventi: .mousedown(), .keypress(), ecc. Se vuoi ascoltare un event per cui jQuery non fornisce un convenience method, puoi usare la function .on():

    $('#search-input').on('input', callback);

Usando un metodo come .click(callback) invece di .addEventListener('click', callback) o .on('click', callback), eviti bug difficili da catturare dovuti a typo di 'click'—invece di far ascoltare al browser eventi 'clik' e non vedere mai risposta alle tue azioni, verra’ riportato un errore (.clik() non e’ una function conosciuta!).

Il parametro event passato alla callback di .click() e’ esattamente come l’event passato alle callback di addEventListener() (anche se jQuery standardizza le differenze cross-browser)—quindi puoi accedere alla sorgente dell’event con la property event.target. Tuttavia, questa property si riferisce a un elemento DOM, non a un selector jQuery. Gli elementi DOM non supportano i metodi jQuery descritti sopra—quelli sono disponibili solo sugli “jQuery selection objects”. Quindi se vuoi lavorare con un elemento DOM, devi “selezionarlo” di nuovo usando la function $(), ottenendo una jQuery selection che ha tutti i metodi utili.

  • Inoltre, jQuery assegna la variable this dentro una event callback al target dell’event; quindi puoi usare $(this) per selezionare l’elemento.

Il capitolo precedente ha citato l’event listener window.onload, usato per determinare quando il DOM ha finito di caricarsi ed e’ pronto per essere manipolato. jQuery fornisce una funzionalita’ simile tramite il listener .ready() (di solito chiamato su tutto il document):

$(document).ready(function() { //non serve che sia una function anonima
    //il programma va qui
    console.log('Ciao mondo!');

    //...
});

Questo e’ un pattern molto comune: spesso con jQuery interi programmi sono scritti dentro la callback .ready(), cosi’ i tag <script> possono stare nel <head> ed essere scaricati rapidamente ma eseguiti solo quando il DOM e’ disponibile. Questo pattern e’ cosi’ comune che la function selector di jQuery puo’ essere usata come scorciatoia:

//equivalente a quello sopra
$(function() {
    //il programma va qui
    console.log('Ciao mondo!');

    //...
});
  • Se passi una function invece di una string Selector alla function jQuery $(), verra’ interpretata come callback da eseguire quando il document e’ pronto! Tienilo presente se guardi codice altrui e vedi che inizia con un $(function) a caso.

E altro!

Questo e’ solo la punta dell’iceberg di cio’ che jQuery puo’ fare. Per esempio, jQuery fornisce anche diverse utility function che possono essere chiamate sul global jQuery (o $):

//controlla se un item e' in un array
$.inArray(4, [3,4,3] );
jQuery.inArray(4, [3,4,3] ); //equivalente, ma forse piu' leggibile

//trova un item in un array che matcha la function di filtro
//questo e' come .filter, ma funziona su browser vecchi (con la versione jQuery giusta)
$.grep( [3,4,3], function(item) {
   return item > 3;
});

//itera su arrays o objects -- funziona per entrambi!
$.each( [3,4,3], function(key, value) {
   console.log('Dammi un '+value);
});

$.each( {first:'Joel',last:'Ross'}, function(key, value) {
   console.log(key+' nome: '+value);
});
  • Questo puo’ essere utile, ma le functions native di JavaScript o i metodi Lodash saranno spesso piu’ semplici e (computationally) piu’ veloci.

E come ultimo esempio divertente: jQuery fornisce anche functions che permettono di creare facilmente animated effects!

$('#id').fadeIn(1000);   //fade in in 1 secondo
$('#id').fadeOut(500);   //fade out in 1/2 sec
$('#id').slideDown(200); //slide down in 200ms
$('#id').slideUp(500);   //slide up in 500ms
$('#id').toggle();       //toggle se visualizzato

//sintassi di animazione custom:
//$(selector).animate({targetProperties}, speed [, doneCallback]);
$("#box").animate({
    left: '500px', //fai volare la box in giro!
    opacity: '0.5',
    height: '200px',
    width: '200px'
}, 1500);
