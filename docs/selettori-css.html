Chapter 6 Selettori CSS

CSS Fundamentals ha introdotto la sintassi base e l’uso di CSS. Questo capitolo dettaglia sintassi aggiuntiva e opzioni per i selector che specificano a quali elementi di una web page si applica una regola CSS.

Ambienti di sviluppo come VS Code possono aiutare a mostrare quali elementi vengono selezionati da una regola a seconda del selector. In VS Code puoi passare il mouse sul selector di una regola CSS e un pop-up ti dara’ un esempio generico di quali elementi quella regola colpisce. Questo e’ utile per verificare la comprensione della sintassi e per confermare che la regola si applichi agli elementi desiderati.

6.1 Selector di base

Quasi tutti i CSS selector sono composti dai tipi piu’ basilari: type selector, class selector o id selector. I type e class selector sono stati introdotti in CSS Fundamentals; le spiegazioni sono ripetute qui per comodita’.

Type Selector

Il selector piu’ basilare e’ il type selector, che seleziona elementi per tipo (nome del tag). Per esempio, la regola sotto si applichera’ a tutti gli elementi <p>, indipendentemente da dove si trovano nella pagina:

p {
   color: purple;
}

Le regole CSS possono usare il type selector per stilizzare qualsiasi tipo di elemento:

/* stilizza tutti gli elementi h1 in viola */
h1 {
  color: purple;
}

/* stilizza tutti gli elementi h2 in oro e italic */
h2 {
  color: gold;
  font-style: italic;
}

/* stilizza tutte le immagini per stare entro una larghezza massima */
img {
  max-width: 300px;
}

Class Selector

Un class selector seleziona tutti gli elementi con un attribute class che contiene il nome specificato. Per esempio, se avessi HTML:

<!-- HTML -->
<p class="highlighted">Questo testo e' evidenziato!</p>
<p>Questo testo non e' evidenziato</p>

Potresti colorare solo il paragraph corretto usando il class selector:

/* CSS */
.highlighted {
   background-color: yellow;
}

I class selector si scrivono con un punto (.) prima del nome della class (non del nome dell’elemento). Il . si usa solo nella regola CSS, non nell’attribute class HTML. Nell’esempio sopra, l’elemento HTML ha la class highlighted (senza punto), e il selector CSS usa .highlighted per dire “elementi con class highlighted”.

I nomi delle class CSS dovrebbero iniziare con una lettera e possono contenere trattini, underscore e numeri. Le parole di solito si scrivono in lowercase e separate da trattini invece che in camelCase o snake_case. Le class dovrebbero avere nomi informativi – come daresti un nome a una variabile. Vedi Naming CSS Classes sotto per ulteriori dettagli.

I class selector si applicano a qualsiasi elemento che contiene quella class, indipendentemente dal tipo dell’elemento. Questo permette di applicare uno styling coerente a tipi diversi di elementi:

<h1 class="alert-flashing">Sono un alert lampeggiante!</h1>
<p class="alert-flashing">Anche io!<p>

Gli elementi HTML possono avere piu’ class; ogni nome di class e’ separato da uno spazio (non da una virgola):

<p class="alert flashing">Ho DUE class: "alert" e "flashing"</p>
<p class="alert-flashing">Ho UNA class: "alert-flashing"</p>
<p class="alert flashing fast">Ho TRE class: "alert", "flashing" e "fast"</p>

Il class selector selezionera’ qualsiasi elemento che contiene quella class nella lista. Quindi il primo paragraph nell’esempio sopra verrebbe stilizzato dal selector .alert o dal selector .flashing. Il secondo paragraph non sarebbe stilizzato da nessuno dei due selector, perche’ “alert-flashing” non e’ la class “alert” o “flashing”.

E’ comune che gli elementi HTML abbiano molte class, specialmente in uno schema modulare di naming.

I class selector sono il modo migliore per stilizzare un singolo elemento (assegnagli una class e poi scrivi una regola che lo selezioni), oltre a essere uno dei modi piu’ chiari ed efficaci per stilizzare grandi porzioni della pagina.

Id Selector

E’ anche possibile selezionare elementi HTML in base al loro attribute id usando un id selector. Ogni elemento HTML puo’ avere un attribute id, ma a differenza di class, il valore dell’id deve essere unico nella pagina. Cioe’, nessun elemento puo’ avere lo stesso id. (Tecnicamente un id selector si applicherebbe a tutti gli elementi con quell’id, ma e’ HTML non valido).

L’id selector si scrive con un carattere #, seguito dal valore dell’id:

<div id="sidebar">
  Questo div contiene la sidebar della pagina
</div>
/* Stilizza l'unico elemento con id="sidebar" */
#sidebar {
  background-color: lightgray;
}

Come per i class selector, il # si usa solo nella regola CSS, non nell’attribute id HTML. Il selector CSS usa #sidebar per dire “l’elemento con id sidebar”.

L’attribute id e’ piu’ specifico (e’ sempre un solo elemento!) ma meno flessibile del class, e rende piu’ difficile riusare lo styling su piu’ elementi o pagine. Quindi e’ best practice usare class o type selector invece degli id selector per lo styling (vedi code style guide). Tuttavia gli id selector sono spesso usati in JavaScript per riferirsi agli elementi (dato che l’identificatore e’ unico).

6.2 Selector complessi

I type selector, class selector e id selector sono i tipi base di selector in CSS. Tuttavia e’ possibile combinare questi tipi base in selector piu’ complessi per specificare regole che si applicano solo a elementi o gruppi particolari. Questa sezione descrive la sintassi per combinare i selector di base.

Grouping Selector

Il grouping selector permette di applicare una singola regola a elementi selezionati da piu’ selector diversi. Tecnicamente non e’ un selector complesso distinto: e’ una “short-cut syntax” per scrivere piu’ regole con selector diversi (ma stesse property) in una volta sola.

Il grouping selector combina i selector con una virgola (,) tra loro:

/* grouping selector: applies to h1, h2, AND h3 elements */
h1, h2, h3 {
    font-family: Helvetica;
    color: #4b2e83; /* UW purple */
}

L’esempio sopra si applica a tutti gli elementi selezionati da un selector h1, da un selector h2 e da un selector h3. E’ esattamente equivalente a scrivere le tre regole separatamente; il grouping selector evita di ripetere le stesse property.

/* Two rules written as one with the grouping selector */
h1, h2 {
  color: #4b2e83;
}

/* Questo e' esattamente lo stesso che scrivere le due regole separatamente */
h1 {
  color: #4b2e83;
}

h2 {
  color: #4b2e83;
}

E’ totalmente accettabile definire la stessa property e value piu’ volte in regole diverse. Non cercare di “minimizzare” le ripetizioni usando il grouping selector. E’ meglio usare il grouping selector solo se semanticamente sta applicando “lo stesso stile”.

I selector componenti nel grouping selector possono essere qualsiasi tipo di selector, inclusi .class, #id o altri selector complessi descritti in questo capitolo:

/* grouping selector can also combine class or id selectors */
/* this rule applies to h2 elements, "menu" classed elements, and the
   id="sidebar" element */
h2, .menu, #sidebar {
    background-color: gray;
}

Nota che, poiche’ le regole successive hanno precedenza sulle precedenti, puoi usare un grouping selector per applicare una property a piu’ elementi, poi aggiungere regole ulteriori per varianti specifiche. Per esempio, puoi avere una regola generale per un’ampia classe di elementi, e regole successive per personalizzazioni specifiche.

/* all headings are Helvetica, bold, and purple */
h1, h2, h3 {
    font-family: Helvetica;
    font-weight: bold;
    color: #4b2e83; /* UW purple */
}

/* h2 elements are also underlined */
h2 {
    text-decoration: underline;
}

Compound Selector

Un compound selector combina selector di elemento, class e id per essere piu’ specifico su dove si applica una regola. Lo fai mettendo un class o id selector immediatamente dopo il selector precedente, senza spazio o virgola tra loro. Il selector si applichera’ agli elementi che corrispondono a tutte le parti componenti.

/* Seleziona solo elementi p che hanno class="alert"
   Altri elementi p e elementi con class "alert" non sono coinvolti */
p.alert {
  color: red;
}

/* Seleziona solo elementi h1 che hanno id="title" */
/* Nota che e' ridondante, dato che solo un elemento puo' avere quell'id! */
h1#title {
  color: purple;
}

/* Seleziona elementi che hanno class "alert" E class "success" */
.alert.success {
  color: green;
  font-size: larger;
}

/* Puoi combinare quanti componenti vuoi */
/* Seleziona elementi che sono `<img>` e hanno class "avatar" E class "small" */
img.avatar.small {
  width: 80px;
  height: 80px;
  border-radius: 50%;
}

/* Puoi combinare con il grouping selector */
/* Seleziona <p class="highlighted"> e <li class="selected"> */
p.highlighted, li.marked {
  background-color: yellow;
}

Nota che mettere uno spazio tra i componenti del selector (ad es. p .alert invece di p.alert) indica un descendant selector (descritto sotto), con un significato completamente diverso. Ogni carattere conta in CSS. Usa strumenti come il pop-up dei selector in VS Code per confermare di aver scritto i selector corretti.

Un compound selector aggiunge specificita’ a un selector, permettendoti di riusare nomi di class (ad es. per semantica condivisa e leggibilita’) ma far applicare stili diversi a elementi diversi. Per esempio, un compound selector permetterebbe a un paragraph “highlighted” (p.highlighted) di apparire diverso da un heading “highlighted” (h1.highlighted). Permette anche di combinare class modulari per personalizzazioni: un alert di successo (.alert.success) puo’ apparire diverso da un alert warning (.alert.warning), con differenze che non si applicano agli elementi .warning generali.

Descendant Selector

I selector menzionati finora si applicano agli elementi corrispondenti indipendentemente dalla loro posizione nell’albero HTML. Ma a volte vuoi essere piu’ specifico e stilizzare solo un insieme di elementi che esistono dentro un particolare parent o ancestor, e non tutti gli elementi altrove nella pagina. Puoi farlo con un descendant selector. Si scrive mettendo uno spazio ( ) tra i componenti del selector. La regola si applichera’ agli elementi che corrispondono all’ultimo componente solo se hanno parent che corrispondono ai componenti precedenti:

<header>
   <h1>Benvenuto nella pagina</h1>
   <p>Sono un paragraph speciale</p>
</header>
<section>
   <p>un altro paragraph</p>
</section>
/*
  Seleziona elementi <p> che esistono dentro elementi <header>
  Altri elementi <p> non saranno coinvolti
 */
header p {
    /* ... */
}

Questo selector si puo’ leggere come “seleziona tutti gli elementi <header>, poi stilizza ogni <p> dentro quegli elementi”. In alternativa, spesso e’ piu’ facile leggere un descendant selector “al contrario”: stilizza gli elementi <p> che sono dentro un <header>.

Un descendant selector puo’ avere quanti “livelli” vuoi, e ogni livello puo’ essere qualsiasi tipo di selector (inclusi compound selector). Tuttavia, e’ buon stile avere solo 2 o 3 livelli. Se ti serve piu’ specificita’, forse e’ il caso di definire una nuova .class.

/* seleziona elementi con class="logo"
   contenuti dentro elementi <p> che a loro volta sono
   contenuti dentro elementi <header> */
header p .logo {
    /* ... */
}

Occhio allo spazio. header p .logo e’ diverso da header p.logo: il primo si riferisce a elementi .logo dentro un <p> (e dentro un <header>), mentre il secondo si riferisce a elementi <p> che hanno la class "logo" (e sono dentro un <header>).

Per chiarezza: i componenti di qualsiasi selector possono essere composti da altri selector:

/* Usa diversi selector in combinazione!
   Stilizza <p> con class "subtitle" dentro un `<header>`, e <legend>
   dentro un <fieldset> dentro un elemento con entrambe le class "sidebar" e
   "form"
   */
header p.subtitle, .sidebar.form fieldset legend { }

Nota che i descendant selector selezionano elementi discendenti a qualsiasi profondita’ nel ramo, non solo child diretti, quindi gli elementi .logo potrebbero essere annidati diversi livelli sotto l’elemento <p> (magari dentro uno <span>). In genere e’ una buona idea perche’ potresti introdurre nuovi livelli di nesting e non vuoi dover modificare il CSS. Ma se vuoi davvero selezionare solo i child diretti, puoi usare una variante chiamata child selector, che usa il simbolo > per indicare solo discendenti diretti:

<body>
  <p>Body content</p>
  <section>
    <p>Section content</p>
  </section>
</body>
/* Seleziona il paragraph "Body content" (child immediato di <body>),
   non il paragraph "Section content" (child immediato di <section>) */
body > p {
  color: blue;
}

Allo stesso modo, il sibling selector (~) seleziona elementi che sono sibling del componente precedente, mentre il adjacent-sibling selector (+) seleziona elementi che sono il sibling immediatamente successivo del componente precedente. Questi selector sono usati piu’ raramente, perche’ dipendono di piu’ dal layout specifico degli elementi e tendono a richiedere aggiornamenti se si aggiungono nuovi contenuti.

/* Sibling Selector: seleziona elementi che sono sibling
   Seleziona *tutti* i <p> che condividono lo stesso parent di un `<img>` e vengono dopo
   */
img ~ p { }

/* Adjacent Sibling Selector: seleziona elementi che sono il sibling *successivo*
   Si applica al singolo <p> che condivide lo stesso parent di un `<img>` e viene
   immediatamente dopo (senza altri elementi in mezzo)
   */
img + p {  }

Pseudo-classi

Una pseudo-class e’ un componente del selector usato per selezionare elementi in base al loro stato corrente di utilizzo, invece che alla loro identita’ esplicita come tipo o class. Puoi pensarle come class predefinite integrate nel browser, che vengono aggiunte o rimosse mentre interagisci con la pagina. Questo permette di definire stili basati sulle interazioni utente (tra le altre cose). Le pseudo-class si scrivono con i due punti (:) seguiti dal nome della pseudo-class, immediatamente dopo un componente del selector (base o complesso):

/* Seleziona elementi <li> su cui lo user passa il mouse
   (solo durante l'hover) */
li:hover {
  font-size: 200%;
}

Per esempio, questa regola si applica a qualunque elemento <li> su cui l’utente passa il mouse, quindi mentre l’elemento e’ in stato “hover” avra’ un font piu’ grande. Quando non e’ piu’ hover, la regola non si applica. Nota che lo styling si applica all’elemento – la pseudo-class descrive solo quali elementi vengono selezionati.

Le pseudo-class possono essere combinate con componenti di selector complessi:

/* Stilizza solo i <li> in hover che sono discendenti di elementi <ul> */
ul li:hover { }

/* Stilizza l'elemento <img> che e' dentro un paragraph in hover */
p:hover img { }

Anche se e’ carino, in generale modificare l’aspetto della pagina con :hover non e’ una grande idea e va usato con parsimonia. :hover non e’ accessibile (uno screen reader non puo’ hoverare). :hover non funziona su dispositivi mobile (non puoi hoverare con il dito). E cambiare lo styling in hover – in particolare se influisce sulle dimensioni – puo’ causare effetti a cascata sul resto della pagina, rendendola piu’ difficile da leggere e usare.

Ci sono molte pseudo-class supportate, che vanno dalle interazioni utente allo stato dei form:

/* Seleziona elementi <input> disabilitati */
input:disabled {
  background-color: grey;
}

/* Seleziona elementi <input> (checkbox e radio button) che sono checked */
input:checked {
  color: green;
}

Alcuni altri tipi comuni di pseudo-class sono descritti nelle sezioni seguenti.

Styling degli hyperlink

Gli elementi hyperlink (<a>) hanno un comportamento di styling di default dipendente dal browser – ad esempio link blu e sottolineati, poi viola se visitati. CSS puo’ sovrascrivere questi stili e usare pseudo-class per specificare lo stile in base allo stato del link:

/* Si applica agli elementi <a> che NON sono stati visitati prima */
a:link {  }

/* Si applica agli elementi <a> che SONO stati visitati prima */
a:visited {  }

/* Si applica agli elementi <a> che hanno focus. Puoi vedere il focus quando usi
   il tasto `tab` per navigare una pagina.
   Particolarmente rilevante per gli screen reader */
a:focus {  }

/* Si applica agli elementi <a> mentre sono "activated" (per esempio, cliccati da uno user) */
a:active {  }

Questi selector possono spesso applicarsi allo stesso elemento: un link puo’ essere non visitato (:link) e avere :focus nel momento in cui viene cliccato (:active). Per assicurare che le regole non vengano sovrascritte, vanno applicate nell’ordine LVHFA: :link, :visited, :hover, :focus, :active. Infatti, spesso le regole vengono scritte per applicarsi a tutti gli stati in quest’ordine:

/* Si applica agli elementi <a> indipendentemente dallo stato */
a:link, a:visited, a:hover, a:focus, a:active {
  ...
}

Ricorda di impostare sempre sia hover sia focus, per supportare l’accessibilita’ delle persone che non possono usare un mouse.

Pseudo-class di struttura dell’albero

Un altro tipo utile di pseudo-class sono quelle che selezionano elementi in base alla loro posizione nell’albero del documento. Servono a scopi simili ai selector discendenti/child/sibling, ma con piu’ capacita’.

Per esempio, le pseudo-class possono selezionare un singolo elemento di un certo tipo in base a quale child e’:

/* Seleziona il primo elemento `<li>` nel suo parent */
li:first-of-type {
  font-weight: bold;
}

/* Seleziona qualunque elemento `<p>` se e' il primo child del suo parent */
/* Se il primo child non e' un `<p>`, non verra' selezionato */
p:first-child {
  font-size: 120%;
}

Queste pseudo-class sono simili ai selector discendenti o child, ma invece di stilizzare un elemento se e’ un child, lo stilizzano se e’ un child particolare. Esistono anche le pseudo-class :last-of-type e :last-child.

Una variante piu’ complessa sono le pseudo-class :nth-of-type e :nth-child, che selezionano elementi in base a un numero o posizione: il “secondo child”, il “quinto child” o “ogni terzo child”.

/* Seleziona il 2o elemento <li> */
li:nth-of-type(2) { }

/* Seleziona ogni elemento <li> "pari" (il conteggio parte da 1) */
li:nth-of-type(even) {}

/* Seleziona ogni terzo elemento <li> */
li:nth-of-type(3n) { }

/* Seleziona ogni terzo elemento partendo dal #2 (quindi 2, 5, 8, ecc.) */
li:nth-of-type(3n+2) { }

Queste sono pseudo-class scritte come funzioni, con un argomento che indica quali elementi selezionare. L’argomento puo’ essere un numero singolo (per selezionare quell’elemento), le keyword even o odd per selezionare ogni altro elemento (contando da 1), o un’espressione An+B, dove A e’ quanti elementi contare e B dove iniziare a contare.

Le pseudo-class strutturali sono utili per elementi ripetuti, come liste o tabelle. Per esempio, puoi usarle per stilizzare “ogni altra riga” o per dare all’ultima riga un aspetto particolare:

/* Seleziona una riga si' e una no (per un aspetto "a righe") */
tr:nth-of-type(even) {
  background-color:#eee;
}

/* Seleziona l'ultima riga (per mettere un bordo piu' spesso in basso) */
tr:last-of-type {
  border-bottom: 5px solid black;
}

Le pseudo-class e altri selector complessi non sono obbligatori quando si scrive CSS. Spesso puoi ottenere lo stesso effetto usando class o combinazioni di selector discendenti. Tuttavia, le pseudo-class possono rendere il codice piu’ espressivo e facile da scrivere e modificare: invece di aggiungere una class a ogni altra riga (e poi aggiornarla se inserisci una riga a meta), puoi usare :nth-of-type(even).

Ci sono molte altre pseudo-class, alcune con argomenti, che possono essere utili per stilizzare elementi specifici. Vedi la documentazione per maggiori dettagli.

Altri selector

Anche se non e’ una lista completa di tutti i selector CSS (vedi ad es. la documentazione MDN), ci sono alcuni tipi di selector aggiuntivi che vale la pena conoscere. Non sono i piu’ usati, ma e’ utile saperli nel caso compaiano in esempi o siti esistenti che stai modificando.

Se trovi un simbolo o un selector in CSS che non capisci, cercalo!

Attribute Selector

L’attribute selector seleziona elementi che hanno un certo attribute. Si scrive mettendo parentesi quadre [] dopo un selector base; dentro le parentesi indichi attribute e valore con sintassi attribute=value:

/* seleziona tutti gli elementi p il cui "lang=sp" */
p[lang="sp"] {
    color: red;
    background-color: orange;
}

E’ possibile selezionare attribute che corrispondono solo parzialmente a un valore; vedi la documentazione degli attribute selector per dettagli.

Nota che e’ piu’ comune usare questo selector per stilizzare input dei form; per esempio, per rendere diverse le checkbox selezionate:

/* seleziona <input type="checkbox"> che hanno lo stato "checked" */
input[type=checkbox]:checked {
    color: green;
}

Non usare gli attribute selector per stilizzare in base a class o id – per quello ci sono class e id selector. Gli attribute selector vanno usati solo quando la presenza dell’attribute e’ la caratteristica che richiede uno styling, come il type di un <input>. Non usarli per valori specifici di attribute generici, come stilizzare un <img> con un certo src o un <a> con un certo href. Questi valori sono fragili e possono cambiare, costringendoti ad aggiornare il selector. Scrivi regole CSS basate sul significato semantico degli elementi (incluse semantiche definite da attribute), non sul contenuto specifico di un elemento. Se vuoi stilizzare un elemento molto particolare, assegnagli una class.

Universal Selector

Anche se raramente usato, e’ possibile selezionare tutti gli elementi della pagina usando il universal selector. Si scrive con * (come un wildcard), e si riferisce a “qualsiasi” elemento:

/* Seleziona tutti gli elementi nella pagina */
* {
  box-sizing: border-box; /* all elements include border and padding in size */
}

/* Combinato con l'Adjacent-Sibling Selector; clear del sibling successivo
  (qualunque esso sia) dopo un elemento floating. Esempio da MDN. */
.floating + * {
  clear: left;
}

Ci sono pochi motivi legittimi per usare questo selector – raramente vuoi applicare una regola a ogni singolo elemento (l’esempio box-sizing sopra e’ un’eccezione). In generale e’ meglio applicare una regola a un elemento di alto livello come <body> e poi affidarsi all’ereditarieta’ per propagare lo stile dove opportuno.

6.3 Specificita’ dei selector

Come notato in CSS Fundamental, poiche’ piu’ regole possono applicarsi a un singolo elemento, e’ possibile che piu’ valori per la stessa property vengano specificati:

/* css */
p { color: blue; }
.alert { color: red; }
<!-- html -->
<p class="alert">
  Questo elemento e' stilizzato da entrambe le regole, entrambe cambiano il `color`. Il testo
  sara' rosso, perche' il selector `.alert` ha specificita' piu' alta. La property della regola `p`
  verra' override.
</p>

Se piu’ regole cercano di specificare la stessa property, allora si applica il value della regola piu’ specifica. Il valore della regola meno specifica si dice overridden. Puoi vedere quali property sono state override nei Chrome Developer Tools – le property override sono barrate. Nota che i valori che non confliggono vengono comunque applicati anche se uno e’ overridden.

La specificita’ di una regola e’ determinata contando il numero di componenti di selector di tipo element, class e id nel selector:

/* Conteggio dei componenti del selector per la specificita' */

p {  } /* 1x type selector */

nav a {  } /* 2x type selector */

ul li a {  } /* 3x type selector */

.highlighted {  } /* 1x class selector  */

p.alert {  } /* 1x class, 1x type selector */

div.alert.success strong {  } /* 2x class, 2x type selector */

#side-nav a {  } /* 1x id, 1x type selector */

/* Gli item in un grouping selector sono contati separatamente! Ogni selector nel
   grouping ha specificita' diversa ed e' applicato separatamente */
.green, p.alert.success {  } /* 1x class selector; 2x class, 1x type selector */

Quindi il numero e la categoria dei componenti di selector in un selector complesso contribuiscono alla specificita’. Gli attribute selector e le pseudo-class contano come class selector.

Ogni categoria di selector e’ contata separatamente, producendo un “punteggio” con 3 numeri: id, class e type. Spesso si scrive come tripla in questo ordine; puoi vederlo in VS Code passando sul selector. Quale regola e’ piu’ specifica dipende da quale ha il conteggio id piu’ alto, poi in caso di pari quale ha il conteggio class piu’ alto, poi in caso di pari quale ha il conteggio type piu’ alto.

/* Selector CSS con conteggi di specificita', ordinati dal piu' specifico al meno specifico */

#side-nav a {  } /* (1,0,1) */

div.alert.success strong {  } /* (0,2,2) */

p.alert {  } /* (0,1,1) */

.highlighted {  } /* (0,1,0) */

ul li a {  } /* (0,0,3) */

nav a {  } /* (0,0,2) */

p {  } /* (0,0,1) */

Importante: ogni categoria (id, class o type) e’ considerata in ordine. Una regola con una sola class (ad es. .highlighted, punteggio (0,1,0)) e’ piu’ specifica di una regola con solo type selector, a prescindere da quanti siano (ad es. nav ul li a em, punteggio (0,0,5)). I punteggi non sono valori posizionali: una regola con 11 class selector avrebbe punteggio (0,11,0).

Le regole di precedenza non sono un motivo per usare #id selector invece di .class o altri combinatori. Scrivi selector con la specificita’ adeguata.

Se due regole sono ugualmente specifiche – hanno lo stesso punteggio – allora si applica l’ultima regola dichiarata, che sovrascrive quella precedente. Le regole CSS sono applicate “in ordine”, con le ultime che override quelle di uguale o minore specificita’.

/* Two rules, both alike in specificity */
p { color: red; }
p { color: blue; }
<p>Questo testo sara' blu, perche' quella regola viene per ultima!</p>

E’ tecnicamente possibile sovrascrivere qualsiasi specificita’ usando la dichiarazione !important. Tuttavia e’ considerata bad practice e va evitata. Puoi sempre scrivere una regola con la specificita’ corretta per ottenere lo styling desiderato.

Anche se la specificita’ e’ importante, in generale non dovrai preoccupartene troppo. La maggior parte del CSS “semantico” e’ scritto con la specificita’ giusta; pensi alle regole a un livello sufficiente fin dall’inizio. Inoltre, le persone tendono a parlare di regole in ordine sempre piu’ specifico, quindi il codice segue quell’ordine. Non avrai spesso regole che si sovrascrivono in modo non intenzionale.

Se noti che una regola di stile non viene applicata pur essendo corretta la sintassi, controlla i developer tools per vedere se la regola e’ sovrascritta da una regola piu’ specifica prima nel foglio (o in un altro foglio). Poi aggiusta il selector in modo che abbia specificita’ uguale o maggiore, ad esempio chiarendo la class parent o il tipo di elemento usando il descendant selector.
