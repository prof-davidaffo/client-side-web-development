Sviluppo Web lato client

Chapter 26 React Native

Come argomento speciale, questo capitolo fornisce una breve introduzione a React Native, un framework per usare React per costruire applicazioni mobile (per esempio, app per Android o iOS). Il framework React Native fornisce un set di build tools che ti permettono di compilare il codice React (scritto in JavaScript) in codice mobile native (Java per Android; Objective-C per iOS). Questo ti permette di usare la tua conoscenza esistente di client-side web development per creare anche mobile apps come se le avessi scritte nel loro normale linguaggio di sviluppo! Inoltre, lo stesso codice React puo’ essere convertito in app sia per iOS che per Android (cosi’ come per il web), dovendo solo aggiustare eventuali feature specifiche di piattaforma. Tre piattaforme al prezzo di una!

  • La spiegazione di questo capitolo rispecchia il tutorial ufficiale di React Native; vedi quello per maggiori dettagli.

26.1 Getting Setup

Il modo piu’ semplice per iniziare una nuova React Native application e’ usare il programma create-react-native-app. Questo funziona quasi in modo identico al programma create-react-app che conosci e ami, tranne che ti fara’ lo scaffolding di una React Native application e fornira’ build scripts usati per testare e sviluppare native apps.

Per usare questo programma installalo globalmente, e poi eseguilo per creare un nuovo React Native project nella cartella corrente:

create-react-native-app MyNativeApp

Questo scaffolding includera’ diversi file di configurazione (vedi la User Guide per i dettagli), ma ti interesserai soprattutto di App.js, il cui export default sara’ il component “root” della tua application (le mobile apps sono di solito progettate attorno a un View component che viene mostrato, invece di uno script stile index.js da eseguire).

create-react-native-app attualmente non funziona con la versione 5.0 o superiore di npm. Fino a quando questo non viene risolto, la soluzione migliore e’ usare yarn per installare e gestire React Native applications. Nota che installare yarn puo’ causare problemi con npm che dovresti risolvere: procedi con cautela se hai una scadenza! In alternativa, puoi fare downgrade a una versione piu’ vecchia di npm usando npm install -g npm@4

Eseguire React Native Apps

Ci sono diversi modi per “eseguire” e testare la tua application mentre la sviluppi, a seconda della piattaforma (Android o iOS) che stai targettando:

  • Per entrambe le piattaforme, puoi testare il tuo codice React Native su un dispositivo fisico usando l’app Expo. Questa e’ una mobile application separata che si “connette” a un server eseguito dai build scripts di create-react-native-app, mostrando gli aggiornamenti alla tua app in tempo reale. Per usare Expo, installa l’app sul tuo telefono, e poi usa npm start per avviare il development server. Il server mostrera’ un QR code nella command line che puoi scansionare per fare in modo che il tuo telefono si connetta al server (assumendo che siano sulla stessa rete wireless); questo eseguira’ la tua app sviluppata sul telefono, e la aggiornera’ automaticamente quando salvi cambiamenti al file!

  • E’ anche possibile eseguire la tua React Native app su virtual devices (per esempio, emulators) per entrambe le piattaforme. Questi sono telefoni “virtuali” che girano sul tuo computer, permettendoti di sviluppare e testare mobile apps senza aver bisogno di un dispositivo specifico.

    Nota che dovrai avere installato e configurato l’ambiente di sviluppo appropriato per ogni piattaforma: Android Studio per Android e Xcode per iOS. Xcode gira solo su MacOS.

    • Per Android, dovrai creare e avviare un emulator con Android Studio: vai su Tools > Android > AVD Manager per aprire l’Android Virtual Device Manager. Poi puoi scegliere “Create Virtual Device…” per avviare il wizard che specifica un nuovo emulator.

      Poi puoi installare ed eseguire un’app sull’emulator (usando Expo) tramite il comando npm run android. Puoi accedere al development menu sul device dalla notifica, o premendo cmd-m.

    • Per iOS, puoi avviare il programma Simulator (si trova dentro Xcode.app/Contents/Developer/Applications. Se fai click destro sul programma Xcode.app e selezioni “Show Package Contents”, potrai navigare fino ad esso). Poi potrai installare ed eseguire un’app su Simulator (usando Expo) tramite il comando npm run ios. Puoi accedere al development menu sul device premendo cmd-d.

  • Infine, Expo Snack ti permette di sviluppare, testare ed eseguire semplici React Native apps interamente online! Questo puo’ essere un buon modo per testare il design o per condividere code snippets con altri.

26.2 React Native Apps

Scrivere codice React Native usa lo stesso processo e le stesse tecniche di scrivere codice React normale: definisci Components che renderizzano Views, che a loro volta sono composte da altri Components! Questi components possono ricevere props e tracciare lo state proprio come in React. Infatti, se guardi il file App.js di default creato da create-react-native-app, vedrai che e’ solo codice React di base.

Tuttavia, invece di renderizzare elementi HTML (come <div> o <button>), le React Native apps renderizzano uno dei built-in components del framework. Ognuno di questi components puo’ essere “compilato” in una versione “native” appropriata. Per esempio, un <View> viene compilato in un elemento <div> sul web, un elemento android.View su Android, e un elemento UIView su iOS.

  • Proprio come i React components restituiscono quasi sempre un <div> con contenuto annidato all’interno, i React Native components restituiscono quasi sempre un <View> con contenuto annidato all’interno. Gli elementi <View> sono particolarmente importanti quando fai styling della tua app; vedi sotto.

Altri components di base includono:

  • I components <Text> rappresentano testo visualizzato (simile nello scopo a un <p>, anche se vengono compilati in elementi <span> inline sul web). Tutto il testo visualizzato deve essere dentro uno di questi elementi; non puoi avere un “text node” direttamente sotto un <View> come puoi in HTML. Gli elementi <Text> supportano anche lo styling a cascata.

  • I components <Image> sono usati per visualizzare immagini (simili a un <img>). Specifichi quale immagine mostrare passando una prop source. Questa puo’ essere un riferimento remoto (per esempio, https://domain.com/picture.png), o un’immagine locale. Per riferirti a un file immagine locale, dovresti importarlo usando il metodo require() di Node, specificando il path al file immagine relativo al Component. Questo permette ai build tools di caricare il file come data URI quando renderizzano l’immagine, e assicura che l’asset sia impacchettato correttamente con la native app.

    <Image source={ require('./path/to/picture.png') }/>

    Inoltre, dovresti sempre specificare una prop accessibilityLabel che indichi come l’immagine dovrebbe essere letta dagli screen readers (si’, esistono anche per dispositivi mobile! Anche le persone non vedenti usano i telefoni).

Styling React Native

Personalizzi l’aspetto dei React Native components specificando properties di styling nella loro prop style, in modo simile al React inline styling. Nota che Android e iOS non supportano CSS, quindi non usi className in React Native.

Tuttavia, poiche’ e’ spesso utile organizzare gli stili in gruppi e dare loro etichette, React Native fornisce un Stylesheet.create() simile a quello usato da Aphrodite. Definire un stylesheet aiuta con l’organizzazione del codice, oltre a fornire un’implementazione native piu’ efficiente (dato che non devi duplicare gli style objects)—e l’efficienza conta molto di piu’ su dispositivi mobile con risorse limitate!

Anche se specifichi properties di stile simili alle properties CSS (per esempio, con gli stessi nomi delle properties), i React Native stylesheets non usano CSS! In particolare, gli stili normalmente non cascadano: specificare fontSize per un <View> non fara’ si’ che quella property venga applicata a piu’ elementi <Text> annidati. Questa feature manca perche’ lo styling normalmente non cascada in Android e iOS, e React Native puo’ girare in modo piu’ efficace non dovendo attraversare l’albero degli elementi per controllare se ogni property e’ definita da un parent. Inoltre, questo significa che ogni component puo’ essere meglio isolato (sviluppato come un pezzo stand-alone e drop-in di un’application), perche’ non c’e’ la possibilita’ di ereditare accidentalmente dello styling.

  • Tuttavia, i components <Text> nested erediteranno dai loro parents per comodita’, permettendoti di stilare facilmente parti di un blocco di testo (per esempio, per evidenziare parte del testo).

Puoi specificare la dimensione di un elemento impostando le properties di stile width e height. Queste properties dovrebbero essere assegnate come numeri senza unita’ (non includi px o rem). Il value misura il numero di density-independent pixels, che e’ un pixel-value che scala in base alla risoluzione (dots-per-inch, o dpi) del dispositivo. Questo permette al sizing di essere consistente su display “retina”.

Qualsiasi elemento che non ha una dimensione fissa—cosi’ come il posizionamento degli elementi—viene gestito principalmente con le properties di Flexbox! Il framework Flexbox ti permette di fornire un layout che sara’ consistente su diverse dimensioni di schermo. L’elemento “root” (di solito un <View>) di un component viene renderizzato come “flex item”, ma ogni <View> puo’ anche essere reso un suo “flex container” per specificare la direzione (che di default e’ una “column” verticale), la dimensione o lo spacing del suo contenuto:

//dichiara uno stylesheet
const styles = StyleSheet.create({
    outer: {
        flex: 1; //questo View dovrebbe riempire lo spazio verticale
        flexDirection: 'row'; //i children dovrebbero essere disposti orizzontalmente
    }
    inner: {
        flex: 1; //prende spazio extra in modo uguale
    }
})

//un'app di esempio
export default class App {
    render() {
        return (
            <View style={styles.outer}>
                <Text style={styles.inner}>Item 1</Text>
                <Text style={styles.inner}>Item 2</Text>
                <Text style={styles.inner}>Item 2</Text>
            </View>
        )
    }
}

Interaction

Le React Native apps possono diventare interattive usando un processo simile alle React apps normali: specifichi un event handler che puo’ essere usato per modificare lo state e re-renderizzare il component. Tuttavia, gli eventi che ascolti sono leggermente diversi con React Native. Per esempio, un elemento <Button> accetta una property onPress (invece di onClick):

<Button onPress={() => this.handlePress()}>Premimi!</Button>
  • Nota che alla callback non vengono passati parametri, quindi non hai un event con cui lavorare.

Per ottenere text input, usi un component <TextInput> (che e’ molto simile a un elemento <input type="text"> in HTML). Questo elemento puo’ essere reso un controlled input proprio come con React normale, anche se useresti la prop onChangeText per ascoltare i cambiamenti di testo (la callback function ricevera’ il testo aggiornato):

<TextInput
    placeholder="Scrivi qualcosa!"
    value={this.state.inputValue}
    onChangeText={ (newText) => this.setState({inputValue:newText}) }
    />

Liste e Data

Le information applications spesso devono mostrare liste di data values (per esempio, una lista di task da completare). Anche se e’ possibile fare map() di una variable array in un array di elementi <View> da renderizzare, Android e iOS supportano tecniche piu’ specifiche che permettono maggiore responsivita’ ed efficienza quando vengono mostrate su dispositivi mobile. Per esempio, questi components renderizzano automaticamente solo una “porzione” della lista che e’ attualmente visibile sullo schermo piccolo, caricando nuove Views in memoria solo quando lo user scorre per vederle. Questo permette allo user di “flickare” in modo fluido una lista di items.

Puoi creare una lista ottimizzata del genere in React Native renderizzando un component <FlatList>. Questo component prende due properties principali: data, che e’ un array di data values da “mappare” in Views; e renderItem, che e’ una function che fa il “mapping” (simile alla function render per react-router):

<FlatList
    data={myDataArray}
    renderItem={ (args) => <Text>{args.item.text}</Text> }
    />

  • Ogni elemento nel data array deve contenere una property key che React usa per tenere traccia di ogni item nella lista (anche se puoi passare una function che estrae un value come key alla prop keyExtractor).

  • Il “data item” effettivo si trovera’ nella property item degli argomenti della callback function (la property index conterra’ l’indice di quell’item). Tuttavia, e’ comune usare object destructuring per passare solo la property specifica dell’object parametro:

    function renderListItem({item}) { //il parametro e' `item = args.item`
    return <Text>{item.text}</Text>; //puoi accedere direttamente a item
}

function renderWithIndex({item, index}) { //ottiene due parametri: args.item e args.index
    return <Text>{index} - {item.text}</TEXT>;
}

Se vuoi scaricare data da internet da mostrare usando React Native, usi l’API fetch() proprio come hai usato sul web!

export default class App extends Component {
    constructor(props){
        super(props);
        this.state = {data:[]}
    }

    componentDidMount() {
        fetch(dataURI)
            .then((res) => res.json())
            .then((data) => {
                this.setState({data: data});
            })
            .catch((err) => console.error)
    }

    render() {
        return (
            <View>
                <FlatList data={this.state.data}
                    renderItem={({item}) => <Text>{item.text}</Text>}
                    />
            </View>
        )
    }
}

In generale, React Native e’ semplicemente un altro modo di costruire (ma non implementare!) React applications, e fa ampio uso di molti framework e tecniche moderne (per esempio, Flexbox, fetch) discussi in questo corso
